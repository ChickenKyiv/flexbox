History

CSS 2.1 defined four layout modes — algorithms which determine the size and position of boxes based on their relationships with their sibling and ancestor boxes:

  • block layout, designed for laying out documents
  • inline layout, designed for laying out text
  • table layout, designed for laying out 2D data in a tabular format
  • positioned layout, designed for very explicit positioning without much regard for other elements in the document

Flexbox layout. Basic terms

Flexbox layout (Flexible box) aiming to provide a more efficient way to create a layout, align and adjust space between items in the container, even if size is unknown or dynamic.

Was designed for layin out more complex applications and webpages

This is why we use term "flex" for this child items. The main idea is to give the ability to alter width and height and order to child elements. It can fit a big part of the process, relate to adjustment web pages for different devices and screens

Note: Flexbox is better to use for components, small sized layouts. While the Grid system is a better choice for large-scale layouts.

What is flexbox?

Flexbox is a set of CSS rules. it involves a lot of things. Flexbox has a hierarchy in CSS rules. Some rules related to a parent components (Flex Container) and child components (Flex Items) Regular CSS blocks use Block + Inline directions. While Flexbox is based on their own directions.

Flexbox Axis

usually items go from MainStart to MainEnd

Main Axis - the primary line along which flex items are laid off. can be horizontal or vertical.

Main Start, Main End - a place where flex items are placed(inside container)

Main Size - depends on what main dimension is. it can be flex item width or height.

Cross-Axis It's an axis perpendicular to the main aixs. it direction depends on main axis directions

Cross-Start, Cross-End - Flex line, filled with items and placed into container.

cross-size - width or height of a flex item......

flex-start: items are packed toward the start line

flex-end: items are packed toward the end line

center: items centered along the line

space-between: items are evenly distributed in the line. first item is on the start line, last item on the end line.

space-around: items are evenly distributed in the line with the equal space on both sides. first item will have one unit of space against to parent container but next item will have 2 units of space between the next item, because that next item has it's own spacing

space-evenly: items are distributed so that the spacing between any two items is equal.

flex-grow

important rule, because now you can be calm about amount of free space or size of flex items inside. Great rule for different screen sizes.

Define ability to grow(if necessary) works as proportion. Number of this rule dictates what amount of the available space inside the flex container the item should take up.

If all items have flex-grow = 1, all remaining space will be distributed equally to all children.

If one item has value 2 -> remaining space would take up twice(increased 2x) as others

Saves a lot of time for Frontend Developers that use it.
